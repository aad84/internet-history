Pylint

It’s Not Just a Linter That Annoys You!

Pylint is a Python static code analysis tool which looks for programming errors, helps enforcing a coding standard, sniffs for code smells and offers simple refactoring suggestions.

Configuring

It’s highly configurable, having special pragmas to control its errors and warnings from within your code, as well as from an extensive configuration file. It is also possible to write your own plugins for adding your own checks or for extending pylint in one way or another.

Free!

It’s a free software distributed under the GNU General Public Licence unless otherwise specified.

Glossary of Terms

Definitions

class - A class is a user-defined blueprint or prototype from which objects are then created. Classes provide a means of bundling data and functionality together. Creating a new class creates a new type of object, allowing new instances of that type to be made.

class method - The @classmethod decorator is a built-in function decorator that is an expression that gets evaluated after the function is defined. The result of that evaluation shadows the function definition. A class method is a method that is bound to the class and not the object of the class.

constructor - Is generally used for instantiating an object. The task of constructors is to assign values to the data members of the class when an object of the class is created. In Python the __init__() method is called the constructor and is always called when an object is created.

design patterns used - one design pattern that we use in our calculator project is the Factory Method which is a Creational Design Pattern. The Factory Method allows an interface or a class to create an object, but lets subclasses decide which class or object to instantiate.

facade - a Structural Design Pattern that provides a simplified, however limited, interface to a complex system of classes, library or framework.

factory - a design pattern used to create concrete implementations of a common interface. It separates the process of creating an object from the code that depends on the interface of the object.

fixture - functions which run before each test function to which it is applied. Fixtures are used to feed some data to the tests such as database connections, URLs to test and some sort of input data.

instantiation - Instantiating a class is creating a copy of the class which inherits all class variables and methods. Instantiating a class in Python is simple. To instantiate a class, we simply call the class as if it were a function, passing the arguments that the __init__ method defines. The return value will be the newly created object.

method - It is a function which is a member of a class, simple as that.

class C:
def my_method(self):
print("I am a C")

c = C()
c.my_method() # Prints("I am a C")

namespace - the constructs used for organizing the names assigned to the objects in a python program. Consider a namespace as a python dictionary.

object - an object is an instance of a class. The object function returns a featureless object which is a base for all classes.

property - The @property decorator is a built-in decorator in Python for the property() function. Use @property decorator on any method in the class to use the method as a property.

static - A static method does not receive an implicit first argument. Static methods are restricted in what data they can access - and they’re primarily a way to namespace your methods.

type cast - a process in which we convert a literal of one type to another. There are two types of type casting, implicit and explicit. Implicit converts data type into another data type automatically. Explicit need user involvement to convert the variable data type into certain data type in order to the operation required.

type hint - a formal solution to statically indicate the type of a value within your Python code.

unit test - The unittest unit testing framework was originally inspired by JUnit and has a similar flavor as major unit testing frameworks in other languages.
